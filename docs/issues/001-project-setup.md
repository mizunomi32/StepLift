# Issue #001: プロジェクト基盤セットアップ

## 概要
StepLiftアプリの開発基盤を整備する。ディレクトリ構成、共通コンポーネント、テーマ設定を実装する。

## タスク

### 1. ディレクトリ構成の作成
以下のディレクトリを作成する：

```
├── components/
│   ├── ui/           # 基本UIコンポーネント
│   ├── workout/      # ワークアウト関連
│   ├── steps/        # 歩数関連
│   └── dashboard/    # ダッシュボード関連
├── lib/
│   ├── db/           # データベース関連
│   ├── stores/       # Zustand ストア
│   ├── hooks/        # カスタムフック
│   └── utils/        # ユーティリティ関数
├── constants/        # 定数
└── types/            # TypeScript型定義
```

### 2. カラーパレットの定義
`constants/colors.ts` に以下のカラーパレットを定義：

| 用途 | ダークモード | ライトモード |
|------|--------------|--------------|
| 背景 | #0A0A0A | #FFFFFF |
| カード背景 | #1C1C1E | #F2F2F7 |
| プライマリ | #22C55E | #16A34A |
| セカンダリ | #3B82F6 | #2563EB |
| アクセント | #F59E0B | #D97706 |
| テキスト | #FFFFFF | #000000 |
| サブテキスト | #9CA3AF | #6B7280 |

### 3. Tailwind設定の拡張
`tailwind.config.js` にカスタムカラーを追加。

### 4. 基本UIコンポーネントの作成
以下のコンポーネントを `components/ui/` に作成：

- `Button.tsx` - 複数バリアント対応のボタン
- `Card.tsx` - カードコンテナ
- `Input.tsx` - テキスト入力
- `Text.tsx` - テーマ対応テキスト

### 5. 依存パッケージのインストール
```bash
pnpm add zustand expo-sqlite
```

## 受け入れ条件
- [ ] ディレクトリ構成が作成されている
- [ ] カラーパレットが定義されている
- [ ] Tailwind設定にカスタムカラーが追加されている
- [ ] 基本UIコンポーネントが作成され、NativeWindでスタイリングされている
- [ ] Zustandがインストールされている
- [ ] 型エラーがない
